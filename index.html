01

What DeSlop checks

Product trust signals

  • Unsupported maturity claims that lack tests, deployment evidence, a threat model, or customer use.
  • Generic product copy that could describe any app instead of the real workflow.
  • Weak error messages that give no cause, no consequence, and no next step.
  • Dead navigation targets that make a demo look more complete than it is.
  • Visible placeholder names and filler that leak into demos.

Implementation residue

  • Unfinished branches and stubbed runtime paths.
  • Artificial demo timers that hide a missing async boundary.
  • Browser-stored auth-like state such as sessions, tokens, roles, or keys.
  • Debug output left outside docs and visible code examples.
  • Unresolved work markers that should become issues, tests, or named non-goals.

Inputs

Markdown, text, JSON, HTML, CSS, JavaScript, TypeScript, JSX, and TSX from a public GitHub repo or local CLI run.

Outputs

Score, severity counts, exact match, file, line, reason, and fix direction for every finding.

Non-goal

DeSlop does not decide whether code was written by a person or a model. It reports visible repo evidence.

02

Run the CLI

# scan the current repo
npx -y github:TheArtOfSound/DeSlop -- .

# fail the build below a score threshold
npx -y github:TheArtOfSound/DeSlop -- . --min-score 90

# machine-readable report
npx -y github:TheArtOfSound/DeSlop -- . --json

# fail on any high-severity finding
npx -y github:TheArtOfSound/DeSlop -- . --fail-on high
# GitHub Actions quality gate
name: DeSlop
on:
  pull_request:
  workflow_dispatch:
jobs:
  deslop:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: TheArtOfSound/DeSlop@main
        with:
          path: .
          min-score: "90"

Scoring is a pressure gauge

The score starts at 100. Each finding subtracts a fixed weight by severity.

high finding−8
medium finding−4
low finding−1

A clean score means the current rules found no configured evidence. It does not prove the product is complete.

04

FAQ

What does DeSlop scan for?

Visible repository evidence: vague product copy, fake production claims, weak error messages, dead navigation, debug residue, browser-stored auth, and unfinished code branches.

Does DeSlop detect whether code was written by AI?

No. DeSlop does not claim to identify who wrote code. It reports concrete repository evidence with file, line, matched text, reason, and fix direction.

Can DeSlop run in CI?

Yes. Run it from the CLI with npx, or wire it as a GitHub Actions quality gate with score and severity thresholds.
