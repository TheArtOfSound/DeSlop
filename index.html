DeSlop is a deterministic browser and CLI repository audit tool. It scans GitHub repos and local projects for concrete signals that software looks unwired, overclaimed, unfinished, or harder to trust.

Inputs

Markdown, text, JSON, HTML, CSS, JavaScript, TypeScript, JSX, and TSX files from a public GitHub repo or local CLI run.

Outputs

Score, severity counts, exact match, file, line, reason, and fix direction for every finding.

Non-goal

DeSlop does not decide whether code was written by a person or a model. It reports visible repo evidence.

What DeSlop checks

Product trust signals

  • Fake maturity claims such as production-ready or enterprise-grade without proof.
  • Generic SaaS copy that could describe any app instead of the real workflow.
  • Weak error messages that give no cause, consequence, or next step.
  • Dead navigation targets that make a demo look more complete than it is.

Implementation residue

  • Unfinished branches and stubbed code paths.
  • Browser-stored auth-like secrets such as passwords, sessions, tokens, or API keys.
  • Debug output left outside docs or visible code examples.
  • Unresolved work markers that should become issues, tests, or named non-goals.

FAQ

What does DeSlop scan for?

DeSlop scans repositories for visible evidence such as vague product copy, fake production claims, weak error messages, dead navigation, debug residue, browser-stored auth, and unfinished code branches.

Does DeSlop detect whether code was written by AI?

No. DeSlop does not claim to identify who wrote code. It reports concrete repository evidence with file, line, matched text, reason, and fix direction.

Can DeSlop run in CI?

Yes. DeSlop can run from the CLI with npx and can be used as a GitHub Actions quality gate with score and severity thresholds.
